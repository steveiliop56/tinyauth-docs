---
title: Configuration
description: Reference on Tinyauth's configuration.
---

Tinyauth can be configured using environment variables or CLI flags. The table below provides a comprehensive list of configuration options.

:::note
  Configuration options with a `FILE_` equivalent (e.g., `USERS` and
  `USERS_FILE`) allow the `FILE_` environment variable or CLI flag to be used as
  an alternative.
:::

## General

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_APPURL` | `--appUrl` | The base URL where the app is hosted. | `` | true |
| `TINYAUTH_RESOURCESDIR` | `--resourcesDir` | The directory where resources are stored. | `./resources` | false |
| `TINYAUTH_DATABASEPATH` | `--databasePath` | The path to the database file. | `./tinyauth.db` | false |
| `TINYAUTH_DISABLEANALYTICS` | `--disableAnalytics` | Disable analytics. | `false` | false |
| `TINYAUTH_DISABLERESOURCES` | `--disableResources` | Disable resources server. | `false` | false |
| `TINYAUTH_DISABLEUIWARNINGS` | `--disableUIWarnings` | Disable UI warnings. | `false` | false |

## Server

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_SERVER_PORT` | `--server.port` | The port on which the server listens. | `3000` | false |
| `TINYAUTH_SERVER_ADDRESS` | `--server.address` | The address on which the server listens. | `0.0.0.0` | false |
| `TINYAUTH_SERVER_SOCKETPATH` | `--server.socketPath` | The path to the Unix socket. | `` | false |
| `TINYAUTH_SERVER_TRUSTEDPROXIES` | `--server.trustedProxies` | Comma-separated list of trusted proxy addresses. | `` | false |

## Auth

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_AUTH_USERS` | `--auth.users` | Comma-separated list of users (username:hash:totp). | `` | true |
| `TINYAUTH_AUTH_USERSFILE` | `--auth.usersFile` | Path to the users file. | `` | false |
| `TINYAUTH_AUTH_SECURECOOKIE` | `--auth.secureCookie` | Enable secure cookies. | `false` | false |
| `TINYAUTH_AUTH_SESSIONEXPIRY` | `--auth.sessionExpiry` | Session expiry time in seconds. | `86400` | false |
| `TINYAUTH_AUTH_SESSIONMAXLIFETIME` | `--auth.sessionMaxLifetime` | Maximum session lifetime in seconds. | `0` | false |
| `TINYAUTH_AUTH_LOGINTIMEOUT` | `--auth.loginTimeout` | Login timeout in seconds. | `300` | false |
| `TINYAUTH_AUTH_LOGINMAXRETRIES` | `--auth.loginMaxRetries` | Maximum login retries. | `3` | false |

## OAuth

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_OAUTH_WHITELIST` | `--oauth.whitelist` | Comma-separated list of allowed OAuth domains. | `` | false |
| `TINYAUTH_OAUTH_AUTOREDIRECT` | `--oauth.autoRedirect` | The OAuth provider to use for automatic redirection. | `` | false |

OAuth configuration includes a provider ID in the environment variable or flag name. For example, `TINYAUTH_OAUTH_PROVIDERS_GOOGLE_CLIENTID`.

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_CLIENTID` | `--oauth.providers.[id].clientId` | OAuth client ID. | `` | true |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_CLIENTSECRET` | `--oauth.providers.[id].clientSecret` | OAuth client secret. | `` | true |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_CLIENTSECRETFILE` | `--oauth.providers.[id].clientSecretFile` | Path to the file containing the OAuth client secret. | `` | false |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_SCOPES` | `--oauth.providers.[id].scopes` | OAuth scopes. | `` | true |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_REDIRECTURL` | `--oauth.providers.[id].redirectUrl` | OAuth redirect URL. | `` | false |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_AUTHURL` | `--oauth.providers.[id].authUrl` | OAuth authorization URL. | `` | true |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_TOKENURL` | `--oauth.providers.[id].tokenUrl` | OAuth token URL. | `` | true |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_USERINFOURL` | `--oauth.providers.[id].userinfoUrl` | OAuth userinfo URL. | `` | true |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_INSECURE` | `--oauth.providers.[id].insecure` | Allow insecure OAuth connections. | `false` | false |
| `TINYAUTH_OAUTH_PROVIDERS_[ID]_NAME` | `--oauth.providers.[id].name` | Provider name in UI. | `` | false |

:::note
  Using `google` or `github` as provider IDs, triggers automatic filling of the
  required information (e.g., auth URLs, scopes). You will only have to
  provide the client ID and secret.
:::

:::note
  Options marked as required are mandatory only when you are the OIDC client.
:::


## OIDC

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_OIDC_PRIVATEKEYPATH` | `--oidc.privateKeyPath` | Path to the private key file. | `./tinyauth_oidc_key` | false |
| `TINYAUTH_OIDC_PUBLICKEYPATH` | `--oidc.publicKeyPath` | Path to the public key file. | `./tinyauth_oidc_key.pub` | false |

OIDC client configuration includes a client ID in the environment variable or flag name. For example, `TINYAUTH_OIDC_CLIENTS_MYAPP_CLIENT_ID`.

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_OIDC_CLIENTS_[ID]_CLIENTID` | `--oidc.clients.[id].clientId` | OIDC client ID. | `` | true |
| `TINYAUTH_OIDC_CLIENTS_[ID]_CLIENTSECRET` | `--oidc.clients.[id].clientSecret` | OIDC client secret. | `` | true |
| `TINYAUTH_OIDC_CLIENTS_[ID]_CLIENTSECRETFILE` | `--oidc.clients.[id].clientSecretFile` | Path to the file containing the OIDC client secret. | `` | false |
| `TINYAUTH_OIDC_CLIENTS_[ID]_TRUSTEDREDIRECTURIS` | `--oidc.clients.[id].trustedRedirectUris` | List of trusted redirect URIs. | `` | true |
| `TINYAUTH_OIDC_CLIENTS_[ID]_NAME` | `--oidc.clients.[id].name` | Client name in UI. | `` | true |

:::note
  Options marked as required are mandatory only when you are the OIDC server.
:::

## LDAP

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_LDAP_ADDRESS` | `--ldap.address` | LDAP server address. | `` | true |
| `TINYAUTH_LDAP_BINDDN` | `--ldap.bindDn` | Bind DN for LDAP authentication. | `` | true |
| `TINYAUTH_LDAP_BINDPASSWORD` | `--ldap.bindPassword` | Bind password for LDAP authentication. | `` | true |
| `TINYAUTH_LDAP_BASEDN` | `--ldap.baseDn` | Base DN for LDAP searches. | `` | true |
| `TINYAUTH_LDAP_INSECURE` | `--ldap.insecure` | Allow insecure LDAP connections. | `false` | false |
| `TINYAUTH_LDAP_SEARCHFILTER` | `--ldap.searchFilter` | LDAP search filter. | `(uid=%s)` | false |
| `TINYAUTH_LDAP_AUTHCERT` | `--ldap.authCert` | Certificate for mTLS authentication. | `` | false |
| `TINYAUTH_LDAP_AUTHKEY` | `--ldap.authKey` | Certificate key for mTLS authentication. | `` | false |
| `TINYAUTH_LDAP_GROUPCACHETTL` | `--ldap.groupCacheTTL` | Cache duration for LDAP group membership in seconds. | `900` | false |

:::note
  For Windows LDAP, use the following search filter: `(&(sAMAccountName=%s))`.
:::

:::note
  Options marked as required are mandatory only when you are using LDAP authentication.
:::

## Log

| Environment | Flag | Description | Default | Required |
| --- | --- | --- | --- | --- |
| `TINYAUTH_LOG_LEVEL` | `--log.level` | Log level (trace, debug, info, warn, error). | `info` | false |
| `TINYAUTH_LOG_JSON` | `--log.json` | Enable JSON formatted logs. | `false` | false |

:::caution
  The `trace` log level will log sensitive information such as usernames, emails
  and access controls. Use with caution.
:::
